Platform System

Product Overview

zett.ai organizes the platform around two building blocks: a server-class RISC-V CPU + programmable NPU compute base, and a CXL/LPU expansion layer for memory, acceleration, storage, and high-speed I/O.

Platform

RISC-V CPU + NPU + CXL/LPU AI Infrastructure

The compute platform combines a 32-core 64-bit RISC-V CPU, DDR5 memory, PCIe Gen5 I/O, virtualization, RAS, hardware security, and a programmable NPU cluster. It is designed as the general compute and AI inference base for cloud, data-center, and local deployment.

The CXL/LPU layer attaches as a programmable acceleration and large-memory expansion node, adding PCIe Gen5 x16, CXL, DDR4 RDIMM, M.2 SSD, and QSFP-DD networking for data movement, memory-resident workloads, and I/O offload.

32-core RISC-V DDR5 6400MT/s PCIe Gen5 x64 64TOPS INT8 CXL + LPU
  • 32-core 64-bit RISC-V CPU, up to 3.0GHz, with single-core SPEC2006 greater than 16/G.
  • 64KB L1 instruction cache, 64KB L1 data cache, 1MB L2 cache, and 32MB L3 cache.
  • Supports RVA23, RISC-V virtualization extensions, AIA-1.0, RVV 1.0, and virtualized DMA access with IOMMU.
  • Four DDR5 subsystems up to 6400MT/s, up to 2TB per subsystem, with ECC and RDIMM/UDIMM/SODIMM support.
  • Four PCIe Gen5 controllers, up to 64 PCIe Gen5 lanes, and up to 32GT/s per lane.

Core Capabilities

Server-Class RISC-V Compute

32-core 64-bit CPU, up to 3.0GHz, a full cache hierarchy, and PCIe Gen5 I/O for cloud compute and high-concurrency workloads.

Large Memory and Fast Interconnects

DDR5 up to 6400MT/s and up to 2TB per subsystem; the CXL/LPU card extends the data path through CXL/PCIe and onboard memory.

AI, Vector, and Matrix Acceleration

The NPU supports INT, FP, and BF16 formats for LLM inference, image generation, vector compute, and matrix operations.

Security, RAS, and Operations

CE/UC/UCE error handling, trusted boot, hardware cryptography, trace/debug, and BMC monitoring support production deployments.

Deployment Use Cases

Where the Platform Fits First

zett.ai is aimed at concrete infrastructure deployments where GPU memory, local inference, and expandable memory capacity directly affect cost, density, and service latency.

GB300 VRAM Expansion

Use CXL/LPU memory expansion to extend the effective memory pool around GB300-class AI systems, reducing memory pressure for large-context inference, retrieval, and memory-resident serving.

Kimi Inference Appliance

Package compute, acceleration, memory, storage, and networking into a deployable Kimi inference system for private, low-latency local serving and controlled data placement.

Standalone Linear-Memory GPU Server

Build standalone GPU servers whose memory capacity can scale linearly through CXL/LPU expansion, improving model hosting density without redesigning the full server platform.

System Architecture

The architecture separates the compute plane, acceleration plane, and data-movement plane, then connects them through PCIe Gen5 and CXL so AI inference, memory expansion, and I/O offload can scale together.
