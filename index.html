Product Overview

zett.ai organizes the platform around RISCV orchestration, programmable XPU acceleration, and a CXL/LPU expansion layer for agent memory, model runtime, storage, and high-speed I/O.

Platform

RISCV + XPU + CXL Agentic Workload Infrastructure

The compute platform combines a server-class RISCV, DDR5 memory, PCIe Gen5 I/O, virtualization, RAS, hardware security, and programmable XPU acceleration. The RISCV provides orchestration, isolation, tool execution, and system control while XPUs handle model, vector, matrix, and data-path work.

The CXL/LPU layer attaches as a programmable acceleration and large-memory expansion node, adding PCIe Gen5 x16, CXL, DDR4 RDIMM, M.2 SSD, and QSFP-DD networking for agent state, retrieval indexes, memory-resident services, and I/O offload.

Server CPU DDR5 6400MT/s PCIe Gen5 x64 XPU Runtime CXL + LPU

Agentic Workload Capabilities

Large Memory for Agent State

Agentic Use Cases

Where RISCV + XPU Agent Platforms Fit First

zett.ai is aimed at generic agentic workload deployments where RISCV scheduling, XPU runtime, large context memory, and isolated tool execution directly affect cost, density, and service latency.

Coding Agent Runtime

RISCV resources schedule containers, file systems, compilers, and tool calls while XPUs run inference, embeddings, code analysis, and reranking against large repo context.

Research and Browser Agents

Package RISCV-managed browsing, retrieval, document parsing, and API calls with XPU-backed summarization, vision, embeddings, and long-context reasoning.

Enterprise Agent Sandbox Pool

Deploy isolated per-project sandboxes with RISCV and XPU budgets, persistent storage, network controls, reproducible runs, and metered usage for teams.

Agentic Workload Architecture

The architecture separates the RISCV orchestration plane, XPU acceleration plane, and data-movement plane, then connects them through PCIe Gen5 and CXL so agent loops, memory expansion, and I/O offload can scale together.
