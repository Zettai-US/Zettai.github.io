Lease Packages

Lease Packages and Use Cases

The program is staged by validation depth. Customers can start with a lower-risk tower or single-node PoC, confirm workload fit, topology, and software stack, then move into a 64-card pilot and rack-level design.

Tower Kit Lease

Developer / lab / customer demo kit

  • • Lanxin CPU/NPU development setup with one GPU or engineering GPU and one CXL FPGA/RPU.
  • • 128GB-512GB CXL memory expansion for KV cache, tensor blocks, or memory-tiering demos.
  • • Best for 30-90 day PoCs, SDK adaptation, driver bring-up, and customer demos.

2U1P Base Node Lease

Single-socket high-density development node

  • • One Lanxin CPU socket, one to two Lanxin NPUs, one CXL FPGA/RPU, and optional B200-class GPU.
  • • Designed for early access, customer PoCs, edge data rooms, private inference, and system software development.
  • • Includes baseline benchmarks, CXL bring-up, NPU runtime checks, and RPU data-path validation.

2U4P High-Density Module Lease

Multi-processor / multi-accelerator module

  • • Four-socket Lanxin CPU or four-node module with multiple NPUs, RPUs, and four to eight GPUs.
  • • 512GB-2TB CXL memory pool for long-context inference, MoE expert weights, and memory-resident services.
  • • Best for 6-12 month pilots, data-center evaluation, and rack-level product definition.

64-Card Pilot Lease

Cluster pilot with reportable metrics

  • • Can be organized as 8x8 GPU, 16x4 mixed-node, or 32x2 high-node-count layouts.
  • • Validates end-to-end value across B200-class GPUs, CXL RPU, Lanxin NPU, CXL memory, and scheduling software.
  • • Produces reports on tokens/s, first-token latency, CXL bandwidth, RPU DMA, NPU offload, and TCO.

Commercial Structure

Suggested Commercial Structure

Public fixed pricing is not recommended. B200-class GPUs, CXL FPGA/RPU, CXL memory, and 2U high-density configurations depend on supply, location, support level, and delivery model, so quotes should be project-based.

PoC lease

30/60/90 day validation for tower kits and 2U1P, including remote support, bring-up checklist, and baseline benchmarks.

Pilot lease

6-12 month pilot for 2U4P and 64-card clusters, including tuning, operations scripts, and milestone reports.

Managed access

Equipment is hosted by zett.ai or a partner facility and accessed through VPN, SSH, or API, reducing customer site power and cooling requirements.

On-prem lease

Equipment is deployed on customer premises after confirming rack, power, cooling, network, security, and service windows.

Qualification

Qualification Before Quote

To avoid hardware mismatch and site deployment failure, each leasing project starts with a short technical qualification.

  1. 1. Workload
    LLM, KV cache, MoE, database, HPC, networking, or storage offload.
  2. 2. Deployment mode
    Managed remote access or on-prem customer deployment.
  3. 3. Accelerator mix
    Quantity and priority of GPU, NPU, RPU, and CXL memory resources.
  4. 4. Site readiness
    Rack, power, cooling, network, VPN, and security policy readiness.
  5. 5. Success metrics
    Tokens/s, latency, HBM usage, CXL bandwidth, and TCO.
  6. 6. Timeline
    30/60/90 day PoC or 6/12/24 month pilot.
